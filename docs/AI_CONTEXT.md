# AI Context: Nine Card Sort プロジェクトについて

## プロジェクトの目的

「ナインカードソート（Nine Card Sort）」を作成し、Web上で公開する。

- 目標: 立ったまま遊べる、手軽でじっくり考えられるカードパズルをデジタル化し、
  ブラウザ上でいつでも遊べるようにする

---

## ゲームのコンセプト・インスピレーション

- **オリジナルコンセプト**: 現実のカードゲームとして設計されたパズル。
  テーブル不要、立ったまま9枚のカードを持ちながら遊べることを前提に設計された。
- **ゲームジャンル**: ソリティア系パズル
- **参考ゲーム**: ソリティア（1人用カードゲームの操作感）

---

## 開発の背景

- 開発者のスキルレベル: Godot未経験・初心者
- GDScriptの経験: なし
- 優先事項: まず動くプロトタイプを完成させること。シンプルな実装を優先する。
- 目標: Web（HTML5）でプレイできる形でリリース

---

## このゲームで重要なポイント

1. **ゲームロジックの正確さが最優先**
   - 山札の配列操作（一番上・一番下・真ん中・N枚目）が正確に動くこと
   - 各カードの能力（9種類）が正確に実装されること
   - 勝利判定が確実に機能すること

2. **カードの選択UIが直感的であること**
   - 能力発動時に「対象のカードを選ぶ」操作が必要な能力がある（1番・2番・3番・9番など）
   - プレイヤーが迷わず対象を選べるUIが必要

3. **参照系能力の挙動が明確であること**
   - 7番・8番は「次のカードの数字X」を参照して動作する
   - Xが1〜9のどの値でも正しく動くこと

---

## ゲームデザイン上の制約・こだわり

- 暴力的・過激な表現は不要。シンプルでクリーンなデザイン
- 1プレイの長さは短め（数分〜10分程度）を想定
- 能力はシンプルに理解できるよう、UIでテキスト説明を表示する
- 操作はクリック（またはタップ）のみで完結させる

---

## AIに期待すること

- GDScriptのコーディングを全面的に担当してほしい
- 山札配列の操作ロジック（挿入・移動・参照）を正確に実装してほしい
- バグを発見・修正してほしい
- 開発者がGodot初心者のため、コードには**必ず日本語コメントを付けてほしい**
- わからない部分は「こういう実装でいいですか？」と確認してから進めてほしい

---

## 人間（開発者）が担当すること

- プレイテストとフィードバック
- カードの能力バランス調整（数値・効果の変更判断）
- ビジュアル・デザインの最終判断
- 各カード能力の最終確定（現時点では仮案）

## 注意事項（開発時に気をつけてほしいこと）

### 山札の配列について

```gdscript
# 山札はArrayで管理。index 0 が「一番上」、index 8 が「一番下」
var deck: Array[int] = [3, 7, 1, 5, 9, 2, 8, 4, 6]
# deck[0] = 3（一番上のカード）
# deck[4] = 9（真ん中のカード）
# deck[8] = 6（一番下のカード）
```

### 能力の実行タイミング

能力は「一番上のカードを一番下に移動させる前」に発動する。
能力発動後、一番上のカードは必ず一番下に移動する（手番処理の一部）。

### 参照系能力（7番・8番）

```gdscript
# 7番・8番の能力は「現在の一番上のカード（能力使用中のカード）の
# 1つ下にあるカード（deck[1]）の数字」を参照する
var x = deck[1]  # 次のカードの数字
```
