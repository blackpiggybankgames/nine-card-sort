## 読むファイル
- docs/temp/fix-bug.md があれば読む（なければ docs/temp/*.md を確認）
- 関連するソースコード
- tests/（関連テスト）

## 手順
1. バグ報告を読み、再現条件を把握
2. 関連するコードとテストを特定
3. 再現テストを書く（RED）
4. 修正を実装（GREEN）
5. リファクタ（必要なら）
6. 全テスト実行
7. 一時ドキュメントを本体に統合し削除(本体というのはFIX_LOG.mdのこと)
   - `cat docs/temp/xxx.md >> 統合先ファイル` で末尾に追記
   - ファイル全体を読み込んで書き直さないこと

## 設計変更を伴う場合
- docs/adr/ に判断記録を残す
- 修正計画を提示し承認を待つ

## 注意
- 計画を提示して人間の承認を待つ。承認なしに実装しない
