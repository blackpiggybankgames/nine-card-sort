## 読むファイル
- `docs/temp/fix-bug.md`（なければ `docs/temp/*.md` を確認）
- 関連するソースコード
- tests/（関連テスト）

## 手順
1. `docs/temp/fix-bug.md` を読み、症状と再現条件を把握
2. 関連するコードを特定
3. 修正計画を判断
   - 設計書に跳ねる変更 → 計画を提示し**人間の承認を待つ**
   - それ以外 → 承認なしで進める
4. 再現テストを書く（テスト実装済みの場合のみ）
5. 修正を実装
6. テスト実行（テスト実装済みの場合のみ）
7. ログに記録・削除
   - `docs/fix-log.md` に修正サマリーを追記（`cat >>` のみ）
   - `rm docs/temp/fix-bug.md`

詳細ルールは `references/workflow.md` を参照。
