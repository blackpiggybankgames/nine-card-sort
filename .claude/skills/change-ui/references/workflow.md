# change-ui 詳細ワークフロー

## ファイル構成と役割

| ファイル | 役割 | このスキルで触るか |
|----------|------|------------------|
| `docs/SYSTEM_SPEC.md` | UI要素・物理パラメータの仕様 | 読むだけ |
| `docs/ui-log.md` | UI変更の追記専用ログ | 追記のみ（全体を読まない） |
| `docs/temp/change-ui.md` | 今回の提案内容 | 読んで、最後に削除 |
| `config/game_balance.json` | 実装値（UIパラメータ含む） | 必要な箇所のみ変更 |

## ui-log.md の追記フォーマット

```
## YYYY-MM-DD | 試行 #N | （変更の一言説明）

変更内容: （何を変えたか）

理由: （なぜ変えたか）

結果: 未評価

---
```

- `cat >> docs/ui-log.md` で追記。ファイル全体を読まない。

## 判断ルール

- レイアウト・配置の大きな変更（見た目・UXに直結） → 選択肢2〜3個提示、人間が決定
- アニメーションの追加・削除 → 選択肢2〜3個提示、人間が決定
- 色・サイズ・速度の微調整 → 計画提示・承認後に実装可

## テストについて

- テストファイルが存在する場合のみ実行
- 存在しない場合はスキップ（ログに「テスト未実装」と記録するだけ）

## エラーパターン

| 状況 | 対処 |
|------|------|
| `change-ui.md` が存在しない | `docs/temp/*.md` を確認。なければ人間に報告 |
| 承認が曖昧 | 「承認」の明示を求める。推測で進めない |
| テストが失敗 | 変更を差し戻し、原因を報告してから再計画 |
