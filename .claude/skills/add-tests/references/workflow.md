# add-tests 詳細ワークフロー

## ファイル構成と役割

| ファイル | 役割 | このスキルで触るか |
|----------|------|------------------|
| `tests/test_*.gd` | テストコード | 新規作成・更新 |
| `tests/.gutconfig.json` | GUT設定ファイル | 初回のみ作成 |
| `scripts/` | テスト対象のソースコード | 読むだけ |
| `docs/temp/add-tests.md` | 今回のテスト追加指示 | 読んで、最後に削除 |

## テスト対象の優先順位

1. **GameState**（ゲーム状態の操作・勝利判定）
2. **能力計算**（各カードの能力ロジック）
3. **バリデーション**（入力検証ロジック）

UIコード・シグナル・描画処理はテスト対象外。

## テスト計画の提示フォーマット

```
## テスト計画

### 対象ファイル
- scripts/game_state.gd → tests/test_game_state.gd

### カバーする機能
- [ ] is_sorted(): 昇順判定
- [ ] apply_ability(): 能力適用後の状態変化
- [ ] ...

承認していただければ実装します。
```

## GUTの設定ファイル（初回作成）

`tests/.gutconfig.json` が存在しない場合は作成する：

```json
{
  "dirs": ["res://tests/"],
  "prefix": "test_",
  "suffix": ".gd",
  "gut_on_load": true
}
```

## 判断ルール

- テスト対象が不明確 → 対象ファイルを列挙して人間に確認
- 既存テストと重複する → 既存テストを更新する（新規作成しない）
- UIコードのテストを求められた → 対象外であることを説明し、ロジック層のテストを提案する

## エラーパターン

| 状況 | 対処 |
|------|------|
| GUTが未導入 | 「addons/gut/ が見つかりません」と報告し、GUT導入を案内して停止 |
| テスト対象コードが副作用を持つ | コードレビューとして指摘し、テスト作成前に人間に報告 |
| テストが失敗 | 失敗内容を報告し、実装バグか テスト側の誤りかを切り分ける |
